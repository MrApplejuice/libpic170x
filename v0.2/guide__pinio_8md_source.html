6 The PinIO library io_control.h allows easy control of a PIC's GPIO pins. It also provides definitions that are internally used by other libraries for interacting with pins. The current set of features is:
7 
8 - Switching pin netween input and output mode
9 - Reading input/output states of individual pins
10 - Writing output values
11 
12 # Reading pin data
13 
14 When a PIC is reset, all pins are set to "input, high impedance mode". Therefore, reading data from individual pins is simply done by invoking the [pin_get_input(PinDef*)](@ref pin_get_input) function. A [PinDef*](@ref PinDef) defines the pin for which the input state should be retrieved. All pins of supported PICs are defines as contants of the form `PIN_RA0`, referring to pin RA0 on a supported PIC. If a pin is not present on a given PIC, like, for example, `PIN_RB4` on a PIC16LF1705, uses of that variable will still be valid but calls to any function in io_control.h for that pin will result in NOP.
15 
16 ~~~~~~~~~~~~~~~~~~~~~~~~~{.c}
17 // ...
18 #include <io_control.h>
19 
20 int main() {
21  bool pin_state = pin_get_input(PIN_RA0);
22  // True will mean that the pin was high, false means the pin was low
23  // NOTE: When reading digital inputs the pin mode should be switched first though! See below.
24 }
25 ~~~~~~~~~~~~~~~~~~~~~~~~~

27 # Switching pin modes
28 
29 To swtich a pin from input to low-impedance output mode, you can use the function [pin_set_pin_mode()](@ref pin_set_pin_mode). The function takes two arguments, the first being the pin to set, the second being the mode, where `true` indicates that the pin should be configured in output mode, and `false` that the pin should be configured in input mode.
30 
31 ~~~~~~~~~~~~~~~~~~~~~~~{.c}
32 
33 // ...
34 #include <io_control.h>
35 
36 int main() {
37  // switch pin RA1 to output-mode
38  pin_set_pin_mode(PIN_RA1, true);
39 }
40 
41 ~~~~~~~~~~~~~~~~~~~~~~~
42 
43 Additionally, pins can be switched between analog and digital input modes. By default all pins are configured to use analog input mode, but for digital signals it is generally advisable to switch pins to ditigal input mode, and even required if an input pin is meant to be used with any other digital subsystem like the UART module. Switching the input mode can be done using the function [pin_set_input_mode()](@ref pin_set_input_mode). The function can set the input mode a pin to:
44 
45 - `PIN_INPUT_MODE_DIGITAL` digital mode
46 - `PIN_INPUT_MODE_ANALOG` analog mode (the default)
47 
48 When intending to read digital input signals, a better example of the first code snipped shown on this page would therefore be:
49 
50 ~~~~~~~~~~~~~~~~~~~~~~~~~{.c}
51 // ...
52 #include <io_control.h>
53 
54 int main() {
55  // First switch to digital input mode before attempting to
56  // read the ditial signal
57  pin_set_input_mode(PIN_RA0, PIN_INPUT_MODE_DIGITAL);
58 
59  bool pin_state = pin_get_input(PIN_RA0);
60  // True will mean that the pin was high, false means the pin was low
61 }
62 ~~~~~~~~~~~~~~~~~~~~~~~~~

64 # Writing pin outputs
65 
66 If a pin has been configured as an output, it is likely that one wnnts to write out data via the pin. This can be achived by using the function [pin_set_output()](@ref pin_set_output). The function takes a single argument, namely the intended output state of the pin.
67 
68 A simple "led blink" example:
69 
70 
71 ~~~~~~~~~~~~~~~~~~~~~~~~~{.c}
72 // ...
73 #include <io_control.h>
74 
75 int main() {
76  // Use RA0 as an output blink an LED on.
77  pin_set_pin_mode(PIN_RA0, true);
78 
79  while (true) {
80  // Toggle output by reading current output and inverting it with '!'
81  pin_set_output(PIN_RA0, !pin_get_input(PIN_RA0));
82  // Wait 100ms to make the blinking visible
83  __delay_ms(100);
84  }
85 }
86 ~~~~~~~~~~~~~~~~~~~~~~~~~
