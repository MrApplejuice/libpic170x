6 The libpic170x modular toolbox that is intended to make using or implemeting certain features of a PIC microcontroller less painful. It either gives a basis to work off, or provides a modular library to build an application from. The library was developed for the [PIC16(L)F1705](https://www.microchip.com/wwwproducts/en/PIC16F1705) and [PIC16(L)F1709](https://www.microchip.com/wwwproducts/en/PIC16F1709) microcontrollers. It is likely that the library will not work with other microcontrollers.
7 
8 Please refer to the [Usage](#mainpage-usage) section for details on how to use the library.

24 # Building/Usage {#mainpage-usage}
25 
26 The library can be used in two different ways:
27 
28 - [As inlined sources](@ref mainpage-as-inline)
29 - [A static library in the `*.lpp` format](@ref mainpage-as-library)

31 Using the library as inlined sources is most likely the most useful way of using the library since the library uses preprocessor macros to compile certain library settings directly into the library to save program memory.

44 ## Use as inlined sources {#mainpage-as-inline}
45 
46 Using the library as inlined sources is most likely the easiest way to use the library. Just copy all the source files for libraries that you would like to use into your your project and compile them yourself. There are not many things to consider. You only need to define `_XTAL_FREQ` to match your desired processor frequency and you are ready to roll!
47 
48 ## Build/use as static library {#mainpage-as-library}
49 
50 Using the library as a static library allows staying up to date with the latest releases of the library easily. However, due to the light-weight setup of the library certain aspects like clock speeds and prescaler factors are directly compiled to binary code and cannot be set when using the library. The values in question specifically are:
51 
52 - Chip type. Supported chips are:
53  - chip=16F1705
54  - chip=16LF1705
55  - chip=16F1709
56  - chip=16LF1709
57 - Base frequency. Supported frequencies are:
58  - xtal_freq=32000000
59  - xtal_freq=16000000
60  - xtal_freq=8000000
61  - xtal_freq=4000000
62  - xtal_freq=2000000
63  - xtal_freq=1000000
64  - xtal_freq=500000
65 
66 To allow users to select the chip type and clock speed when setting up a project the compiled library name includes the parameter selection for `chip` and `frequency` in its name. Example:
67 
68 ~~~~~~~~~~~~~~~~~~~~~~~~
69 libpic170x_16LF1705_2000000.lpp
70 ~~~~~~~~~~~~~~~~~~~~~~~~
71 
72 This library is compiled for a PIC16LF1705 micro controller running its main processor clock at 2 MHz.
73 
74 The sources for `libpic170x` contain a convenience bash-script with the name `build_all.sh` which will build all supported combinations of chip and frequencies in one go. Alternatively, if only one parameter set must be built, it is possible to control which library is being built by defining the make variables `chip` and `xtal_freq` and setting them to the apropriate parameters. Example: `make chip=16LF1705 xtal_freq=8000000` will build `libpic170x_16LF1705_8000000.lpp`.
75 
76 ### Project compile options
77 
78 After the library is built, it will be placed in the install-directory of the project. Configure your PIC-project as follows:
79 
80 - Add _XTAL_FREQ compiler definition matching the library build flags.
81 - Add include directory `install/include` to your project
82 - Add the correct link library for your settings from the `install/` directory
83 
84 Then your are set. For a working example, check the blink-demo application from the examples-directory.
